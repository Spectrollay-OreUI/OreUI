加载中 OreUI 设计
引言 本项目界面风格灵感源于经典像素视觉范式. 除风格参考外, 与 Mojang AB 及其开发的同名软件资产无任何直接或间接关联, 亦不包含任何官方专有代码及资产. 什么是 OreUI 设计系统？ OreUI 设计系统是一套由规范、组件与工具构成的完整体系。它为开发者、设计师、测试人员以及更多相关角色在构建项目和其他数字产品时提供创作流程上的指导。 组件是构建界面的基础单元，具备足够的灵活性，可适用于多种不同的需求与交互行为。通过对组件进行组合，可以构建新的功能、界面以及用户流程。 规范则用于帮助团队在实际开发过程中，对组件的使用方式与适用场景做出一致且合理的设计决策。 如何使用 在设计系统中，你可以通过浏览文档访问当前所有可用的规范、组件与工具。每个组件都附带对应的使用规范，应遵循这些规范以确保最佳实践、可访问性以及最佳用户体验。 组件状态会显示在各组件页面以及组件概览页面中。我们建议优先使用处于稳定 (Stable) 状态的组件。 社区 (Community) 与实验性 (Experimental) 组件同样可以使用，但请注意其可能存在一定风险，例如不支持所有平台、后续可能发生较大变更等。 已弃用 (Deprecated) 组件将会逐步移除，请勿继续使用。 贡献 如果你希望参与本项目的开发、改进或维护，可以通过项目仓库中的相关联系方式与我们取得联系，我们会提供必要的信息并帮助你快速了解项目结构与开发流程。 本项目欢迎任何形式的贡献，包括代码优化、问题修复、文档完善、设计建议以及功能改进等内容。你也可以通过 GitHub Issue 提交问题反馈或功能建议，或 Fork 仓库后提交 Pull Request 参与开发。 如果你希望进一步了解项目的模块组织结构、组件设计理念以及本地开发环境的搭建方式，请查阅相关开发文档与使用指南。 如何反馈漏洞 如果你在使用组件、浏览文档或运行项目时发现问题，可以通过 GitHub Issue 提交详细的问题反馈。为了帮助我们更快定位问题，建议尽可能提供复现步骤、运行环境、截图或相关日志信息。 对于功能建议、技术探讨或疑难解答等内容，也可以通过项目页面中的联系方式与我们进行交流与反馈。 V {VERSION} MIT Licensed | © 2020 Spectrollay
